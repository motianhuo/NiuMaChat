# 职能沟通翻译助手

一个基于 AI 的协作工具，旨在消除产品经理（PM）与开发工程师（Dev）之间的“语言隔阂”。

## 核心功能

- **产品 ➔ 开发 翻译**: 将业务需求转化为技术语言，补充算法、数据处理、性能指标及工作量预估。
- **开发 ➔ 产品 翻译**: 将技术成果转化为业务价值，强调用户体验、增长空间和商业效率。
- **智能角色识别**: 自动检测输入内容所属角色，实现一键翻译。
- **流式输出**: 实时展示 AI 生成过程，提升交互体验。

## 快速开始

### 1. 安装依赖

确保已安装 Python 3.8+，然后在项目目录下运行：

```bash
pip install -r requirements.txt
```

### 2. 配置 API Key

你可以通过以下两种方式配置：
- **方式 A (推荐)**: 在项目根目录创建 `.env` 文件，内容如下：
  ```env
  OPENAI_API_KEY=你的API_KEY
  OPENAI_BASE_URL=https://api.openai.com/v1
  ```
- **方式 B**: 启动应用后，在页面侧边栏手动输入 API Key。

### 3. 运行项目

```bash
streamlit run app.py
```

## 测试用例

### 用例 1：产品视角输入
**原始内容**: `我们需要一个智能推荐功能,提升用户停留时长`
**翻译方向**: 产品经理 ➔ 开发工程师
**翻译重点**: 推荐系统架构、冷启动方案、召回与排序算法、吞吐量要求。

### 用例 2：开发视角输入
**原始内容**: `我们优化了数据库查询，QPS提升了30%`
**翻译方向**: 开发工程师 ➔ 产品经理
**翻译重点**: 页面加载速度提升对转化率的贡献、支持更高并发下的用户增长。

### 用例 3：运营/管理层目标
**原始内容**: `项目目前进度落后2周，主要是因为第三方API对接不稳定，需要协调资源解决。`
**翻译方向**: 翻译给 ➔ 运营/管理层
**翻译重点**: 风险量化（延期对业务的影响）、成本收益比、资源协调的优先级建议。

## 提示词设计思路

1. **角色锚定**: 
   - 翻译给开发时，AI 扮演“资深技术架构师”，强调严谨性、细节和实现路径。
   - 翻译给产品时，AI 扮演“产品专家”，强调用户价值、商业语言和结果导向。
2. **主动补充**: 提示词要求 AI 在 PM 描述模糊时，主动提出技术上必须关注的边界条件（如并发量、安全性等）。
3. **术语对齐**: 针对不同角色使用其领域内的特定词汇（如：QPS vs 留存率）。
